Grow Detail — Layout Approaches

Welches Layout fühlt sich für Mobile-first richtig an?

A

Feed-First

Header → Pflanzen-Scroller → Timeline-Feed. Chronologische Timeline als Hauptinhalt, Pflanzen als kompakte horizontal scrollbare Karten.

B

Zusammenfassung-Header

Dichter Status-Header mit Grow-Status, Pflanzen-Phase, letzte Aktivität auf einen Blick. Dann Pflanzen-Karten und Timeline darunter.

C

Tabbed Sections

Tabs: Übersicht | Pflanzen | Tagebuch | Erinnerungen. Jeder Tab zeigt nur das Wesentliche, kein Scrollen durch gemischte Inhalte.

Vorteile

  • A (Feed-First): Schnellste Überblick — Timeline zeigt sofort alle Aktivitäten
  • B (Summary): Sofortiger Status auf einen Blick, vertraut von anderen Apps
  • C (Tabs): Keine Informationsüberflutung, jeder Bereich fokussiert

Nachteile

  • A: Pflanzen brauchen extra Scroll, Status nicht oben
  • B: Kann bei vielen Pflanzen/Data overflooden
  • C: Navigation zwischen Tabs ist langsamer
