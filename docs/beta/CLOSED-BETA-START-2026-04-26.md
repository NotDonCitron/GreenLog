## Feedback Process
1. Testers receive invite + scope rules.
2. Testers report issues with:
   - route/screen
   - device/browser
   - reproduction steps
   - screenshot/screen recording if possible
3. Team triages into:
   - critical (blocking)
   - major (high friction)
   - minor (polish)
4. Fixes are validated on preview/production before next tester expansion.
