## New Endpoints to Create

1. **POST /api/organizations/[organizationId]/membership-request**
   - Authenticated user requests to join
   - Check if org requires approval
   - If yes: create membership with status `pending`
   - If no: create membership with status `active` immediately

2. **PATCH /api/organizations/[organizationId]/members/[memberId]/approve**
   - Admin approves a pending member
   - Update membership_status from `pending` to `active`
   - Set joined_at to now()
   - Write activity: member_approved

3. **PATCH /api/organizations/[organizationId]/members/[memberId]/reject**
   - Admin rejects a pending member
   - Update membership_status to `rejected`
   - Store rejection_reason
   - Write activity: member_rejected

  <action>
Create POST endpoint for requesting organization membership.

**Route:** `POST /api/organizations/[organizationId]/membership-request`

**Logic:**
1. Authenticate user (required)
2. Check if user is already a member (active or pending) - return 409 if so
3. Fetch organization to check `requires_member_approval` flag
4. If user is invited (status=invited), they can accept — skip approval flow
5. Create membership record:
   - If `requires_member_approval = true`: status = `pending`, joined_at = null
   - If `requires_member_approval = false`: status = `active`, joined_at = now()
6. Return success with membership status

**Response shapes:**
- 201: `{ data: { membership, requires_approval: boolean }, error: null }`
- 400: User already has pending request
- 409: User already a member
- 404: Organization not found

Use the existing authenticateRequest pattern and jsonSuccess/jsonError helpers.
</action>

7. Return updated membership

**Response shapes:**
- 200: `{ data: { member }, error: null }`
- 403: Not admin or target is not pending
- 404: Member not found
- 400: Cannot approve non-pending membership
