## Optimistic Pattern (from useCollection.ts lines 102-121)
```typescript
onMutate: async () => {
  await queryClient.cancelQueries({ queryKey: ["collection-ids", user?.id] });
  const previousIds = queryClient.getQueryData<string[]>(["collection-ids", user?.id]);
  queryClient.setQueryData<string[]>(["collection-ids", user?.id], optimisticUpdate);
  return { previousIds };
},
onError: (_err, _vars, context) => {
  if (context?.previousIds) {
    queryClient.setQueryData<string[]>(["collection-ids", user?.id], context.previousIds);
  }
},
onSettled: () => {
  queryClient.invalidateQueries({ queryKey: ["collection", user?.id] });
},
```

Note: FollowButton does not use queryClient.setQueryData for optimistic status (unlike useCollection). Instead, it uses local state. The optimistic update for FollowButton should:
1. Snapshot current computedStatus before mutation
2. Immediately update local UI state on click
3. On error: revert to snapshot, show alert
4. On success: invalidation still happens (already implemented)
